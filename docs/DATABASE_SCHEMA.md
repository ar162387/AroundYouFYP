# Database Schema — Local PostgreSQL

> **Purpose:** Defines the clean, self-hosted PostgreSQL schema for the Ay backend.
> This replaces the Supabase-managed schema. AI-related tables and Supabase-specific
> system tables are excluded. All `auth.users` foreign keys are replaced with
> references to `AspNetUsers` (ASP.NET Core Identity table).
>
> **Extensions Required:**
> ```sql
> CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
> CREATE EXTENSION IF NOT EXISTS postgis;
> ```

---

## Entity Relationship Overview

```
AspNetUsers (Identity)
    │
    ├──► user_profiles          (1:1 — always created on register)
    │
    ├──► merchant_accounts      (1:1 — created when user applies as merchant)
    │         │
    │         └──► shops        (1:N — merchant owns multiple shops)
    │                   │
    │                   ├──► merchant_categories     (1:N)
    │                   │         └──► merchant_item_categories  (N:M junction)
    │                   │
    │                   ├──► merchant_items           (1:N)
    │                   │         └──► merchant_item_categories  (N:M junction)
    │                   │         └──► item_templates            (N:1 optional)
    │                   │
    │                   ├──► shop_delivery_logic      (1:1 — auto-created on shop insert)
    │                   ├──► shop_delivery_areas      (1:N)
    │                   ├──► delivery_runners         (1:N)
    │                   ├──► audit_logs               (1:N)
    │                   └──► orders                  (1:N — orders placed at this shop)
    │
    ├──► consumer_addresses     (1:N — consumer's saved addresses)
    │
    └──► orders                 (1:N — orders placed by this user)
              │
              ├──► order_items          (1:N)
              ├──► delivery_runners     (N:1 optional)
              ├──► consumer_addresses   (N:1 — snapshot FK)
              └──► reviews             (1:1 optional)

AspNetUsers
    ├──► device_tokens              (1:N)
    └──► notification_preferences   (1:1)
         └──► notification_audit_log (via order_id)
```

## Schema Definitions

> **Convention:**
> - All primary keys are `UUID` generated by `gen_random_uuid()` or EF Core `Guid.NewGuid()`
> - All timestamps are `TIMESTAMPTZ` (timestamp with time zone), defaulting to `NOW() AT TIME ZONE 'utc'`
> - All enum columns are stored as `TEXT` with a `CHECK` constraint; EF Core uses value converters
> - `AspNetUsers.Id` is `UUID` (custom Identity setup with `IdentityUser<Guid>`)

CREATE TABLE user_profiles (

**`opening_hours` JSONB structure:**
```json
{
  "monday":    { "open": "09:00", "close": "22:00", "is_open": true },
  "tuesday":   { "open": "09:00", "close": "22:00", "is_open": true },
  "wednesday": { "open": "09:00", "close": "22:00", "is_open": true },
  "thursday":  { "open": "09:00", "close": "22:00", "is_open": true },

### Table: `shop_delivery_logic`

```sql
CREATE TABLE shop_delivery_logic (
    id                       UUID          NOT NULL DEFAULT gen_random_uuid(),
    shop_id                  UUID          NOT NULL UNIQUE,  -- FK → shops.id
    minimum_order_value      NUMERIC       NOT NULL DEFAULT 200.00 CHECK (minimum_order_value > 0),
    small_order_surcharge    NUMERIC       NOT NULL DEFAULT 40.00  CHECK (small_order_surcharge >= 0),
    least_order_value        NUMERIC       NOT NULL DEFAULT 100.00 CHECK (least_order_value > 0),
    distance_mode            TEXT          NOT NULL DEFAULT 'auto' CHECK (distance_mode IN ('auto', 'custom')),
    max_delivery_fee         NUMERIC       NOT NULL DEFAULT 130.00 CHECK (max_delivery_fee > 0),
    distance_tiers           JSONB         NOT NULL DEFAULT '[
        {"max_distance": 200,  "fee": 20},
        {"max_distance": 400,  "fee": 30},
        {"max_distance": 600,  "fee": 40},
        {"max_distance": 800,  "fee": 50},
        {"max_distance": 1000, "fee": 60}
    ]',
    beyond_tier_fee_per_unit NUMERIC       NOT NULL DEFAULT 10.00  CHECK (beyond_tier_fee_per_unit >= 0),
    beyond_tier_distance_unit NUMERIC      NOT NULL DEFAULT 250.00 CHECK (beyond_tier_distance_unit > 0),
    free_delivery_threshold  NUMERIC       NOT NULL DEFAULT 800.00 CHECK (free_delivery_threshold >= 0),
    free_delivery_radius     NUMERIC       NOT NULL DEFAULT 1000.00 CHECK (free_delivery_radius >= 0),
    created_at               TIMESTAMPTZ   NOT NULL DEFAULT NOW() AT TIME ZONE 'utc',
    updated_at               TIMESTAMPTZ   NOT NULL DEFAULT NOW() AT TIME ZONE 'utc',
    CONSTRAINT shop_delivery_logic_pkey     PRIMARY KEY (id),
    CONSTRAINT shop_delivery_logic_shop_fk  FOREIGN KEY (shop_id) REFERENCES shops(id) ON DELETE CASCADE
);
```

**`distance_tiers` element structure:** `{ "max_distance": <meters>, "fee": <PKR> }`

**Business Rules (enforced in Application layer):**
- `least_order_value` — Absolute minimum; order rejected if below
- `minimum_order_value` — If subtotal < this, `small_order_surcharge` is added
- Distance tiers are sorted ascending and matched by `distanceInMeters <= tier.max_distance`
- Beyond last tier: `fee = lastTier.fee + ceil((distance - lastTier.max_distance) / beyondTierDistanceUnit) * beyondTierFeePerUnit`
- Capped at `max_delivery_fee`
- If `subtotal >= free_delivery_threshold AND distance <= free_delivery_radius` → fee = 0

### Table: `orders`

```sql
CREATE TABLE orders (
    id                         UUID        NOT NULL DEFAULT gen_random_uuid(),
    order_number               TEXT        NOT NULL UNIQUE,
    shop_id                    UUID,                  -- FK → shops.id
    user_id                    UUID        NOT NULL,  -- FK → AspNetUsers.Id (consumer)
    consumer_address_id        UUID        NOT NULL,  -- FK → consumer_addresses.id
    delivery_runner_id         UUID,                  -- FK → delivery_runners.id (assigned on dispatch)
    status                     TEXT        NOT NULL DEFAULT 'pending'
                               CHECK (status IN ('pending', 'confirmed', 'out_for_delivery', 'delivered', 'cancelled')),
    subtotal_cents             INTEGER     NOT NULL CHECK (subtotal_cents >= 0),
    delivery_fee_cents         INTEGER     NOT NULL DEFAULT 0 CHECK (delivery_fee_cents >= 0),
    surcharge_cents            INTEGER     DEFAULT 0 CHECK (surcharge_cents >= 0),
    total_cents                INTEGER     NOT NULL CHECK (total_cents >= 0),
    payment_method             TEXT        NOT NULL DEFAULT 'cash'
                               CHECK (payment_method IN ('cash', 'card', 'online')),
    special_instructions       TEXT,
    -- Timestamp milestones
    placed_at                  TIMESTAMPTZ NOT NULL DEFAULT NOW() AT TIME ZONE 'utc',
    confirmed_at               TIMESTAMPTZ,
    out_for_delivery_at        TIMESTAMPTZ,
    delivered_at               TIMESTAMPTZ,
    cancelled_at               TIMESTAMPTZ,
    -- Derived time metrics (seconds) — computed in Application layer on status transition
    confirmation_time_seconds  INTEGER     CHECK (confirmation_time_seconds >= 0),
    preparation_time_seconds   INTEGER     CHECK (preparation_time_seconds >= 0),
    delivery_time_seconds      INTEGER     CHECK (delivery_time_seconds >= 0),
    -- Cancellation
    cancellation_reason        TEXT,
    cancelled_by               UUID,       -- FK → AspNetUsers.Id
    -- Snapshots (immutable at time of order)
    delivery_address           JSONB       NOT NULL,
    customer_name              TEXT,
    customer_email             TEXT,
    customer_phone             TEXT,
    created_at                 TIMESTAMPTZ NOT NULL DEFAULT NOW() AT TIME ZONE 'utc',
    updated_at                 TIMESTAMPTZ NOT NULL DEFAULT NOW() AT TIME ZONE 'utc',
    CONSTRAINT orders_pkey              PRIMARY KEY (id),
    CONSTRAINT orders_user_fk           FOREIGN KEY (user_id) REFERENCES "AspNetUsers"("Id"),
    CONSTRAINT orders_shop_fk           FOREIGN KEY (shop_id) REFERENCES shops(id),
    CONSTRAINT orders_address_fk        FOREIGN KEY (consumer_address_id) REFERENCES consumer_addresses(id),
    CONSTRAINT orders_runner_fk         FOREIGN KEY (delivery_runner_id) REFERENCES delivery_runners(id),
    CONSTRAINT orders_cancelled_by_fk   FOREIGN KEY (cancelled_by) REFERENCES "AspNetUsers"("Id")
);
CREATE INDEX idx_orders_user_id    ON orders(user_id);
CREATE INDEX idx_orders_shop_id    ON orders(shop_id);
CREATE INDEX idx_orders_status     ON orders(status);
CREATE INDEX idx_orders_placed_at  ON orders(placed_at DESC);
```

**`delivery_address` JSONB structure:**
```json
{
  "id": "uuid",
  "title": "home",
  "street_address": "123 Main St",
  "city": "Karachi",
  "region": "Sindh",
  "latitude": 24.8607,
  "longitude": 67.0011,
  "landmark": "Near mosque",
  "formatted_address": "123 Main St, Karachi"
}
```

**Status transition machine (enforced in Application layer):**
```
pending → confirmed         (merchant action)
pending → cancelled         (consumer OR merchant — only from pending)
confirmed → out_for_delivery (merchant action — requires runner assignment)
out_for_delivery → delivered (merchant action)
out_for_delivery → cancelled (merchant action — not supported in v1)
delivered → (terminal)
cancelled → (terminal)
```

**Time metrics:** Calculated and written on each transition:
- `confirmation_time_seconds` = `confirmed_at - placed_at` (seconds)
- `preparation_time_seconds` = `out_for_delivery_at - confirmed_at`
- `delivery_time_seconds` = `delivered_at - out_for_delivery_at`

    changed_fields     JSONB       NOT NULL DEFAULT '{}',
    source             TEXT        NOT NULL DEFAULT 'manual',  -- 'manual' | 'system'
    created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW() AT TIME ZONE 'utc',
    CONSTRAINT audit_logs_pkey     PRIMARY KEY (id),
    CONSTRAINT audit_logs_shop_fk  FOREIGN KEY (shop_id) REFERENCES shops(id) ON DELETE CASCADE,
    CONSTRAINT audit_logs_item_fk  FOREIGN KEY (merchant_item_id) REFERENCES merchant_items(id) ON DELETE SET NULL
);
```

## Order Number Generation

Supabase used a PostgreSQL sequence and `generate_order_number` function. The .NET equivalent uses a PostgreSQL sequence with an `IOrderNumberGenerator` service:

```sql
CREATE SEQUENCE order_number_seq START 1000 INCREMENT 1;
```

```csharp
// In IOrderNumberGenerator implementation
var seq = await _context.Database
    .SqlQuery<long>($"SELECT nextval('order_number_seq')")
    .FirstAsync();
return $"AY-{seq:D6}";  // AY-001000
